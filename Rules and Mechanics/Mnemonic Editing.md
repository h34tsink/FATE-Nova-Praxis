# Mnemonic Editing

Mnemonic editing allows a trained character to repair a damaged ego, similar to debugging complex computer code. This process involves making changes to the intricate code of the mind—a potentially dangerous procedure.

- **Limitations:** Specific memories cannot be targeted for erasure, nor can personality traits simply be removed. However, areas indicating psychological trauma can be isolated and dampened. The memory remains intact, but the traumatic reaction is lessened, helping the patient come to terms with the event.

## Game Mechanics

- The patient must be in a deactivated SIM state, and their ego must be hosted on a system with hostware.
- The character performs a **Mnemonics Skill test** against a difficulty set by the type of Consequence to be downgraded:

| Consequence | Difficulty |
|-------------|------------|
| Mild        | 4          |
| Moderate    | 5          |
| Severe      | 6          |
| Extreme     | 7          |

- **Success:** The targeted Consequence is reduced as if it had recovered naturally. (See page 123 for more on recovering Consequences.)
- **Failure:** Nothing happens; the patient’s status does not change.
- **Stall:** The patient suffers mental stress equal to the amount by which you failed.

## Example Use

- A character attempts to lessen the trauma of a severe event. They make a Mnemonics Skill test against Difficulty 6. If successful, the patient's Severe Consequence is downgraded.
- If the test stalls, the patient suffers additional mental stress.

> **GM Tip:** Use mnemonic editing as a way to roleplay therapy, recovery, or technical intervention scenes. Highlight the risks and emotional stakes involved.
