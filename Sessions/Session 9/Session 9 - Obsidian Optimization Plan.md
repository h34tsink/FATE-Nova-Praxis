# Session 9: Obsidian Optimization Plan

Goal: run Session 9 faster at the table, with less tab-hunting and less duplicated prep.

## Current Gaps (Audit)

- Runtime guidance was split across multiple long notes with overlap.
- Critical play controls (exposure, merge outcome, nowak race) were not centralized in one operational board.
- Some files are deep reference and others are runtime, but that boundary was not explicit.

## New Operating Model

- **Command center:** [[Session 9 - Ops Index]]
- **Runtime authority:** [[Session 9 - Guide]]
- **Scene kit:** [[Session 9 - Scenes and Zones]]
- **Deep fallback:** [[Session 9 - Beats (GM Runtime)]] and Valare references
- **Live tracking:** [[Session 9 - GM Command Board]]

## Feature Recommendations (You Already Have Most Installed)

### 1) Kanban (Installed)

Use [[Session 9 - GM Command Board]] as your live control surface.

- Benefit: visual state changes during play
- Best use: scene queue + exposure + merge outcome + unresolved hooks

### 2) Dataview (Installed)

Add a session control query note later to auto-list open hooks, unresolved NPC debts, and heat markers.

Suggested data fields to standardize in future session notes:

- `session:`
- `status:` (prep/live/done)
- `heat:` (none/low/med/high)
- `open_loops:`

### 3) QuickAdd (Installed)

Create one-click captures during play:

- "Add Consequence"
- "Add New Hook"
- "Log Faction Shift"

This prevents losing in-session improvisation details.

### 4) Templater (Installed)

Create micro-templates for:

- scene handoff line
- compel prompt
- consequence entry
- post-session debrief block

### 5) Workspaces Plus (Installed)

Create a dedicated workspace named **Session 9 Live** with:

- Pane A: [[Session 9 - Guide]]
- Pane B: [[Session 9 - GM Command Board]]
- Pane C: [[Session 9 - Scenes and Zones]]
- Pane D: [[Session 9 - Beats (GM Runtime)]]

### 6) Supercharged Links + Properties (Installed)

Use properties in session notes to visually classify:

- `phase: prep|live|post`
- `focus: investigation|negotiation|civil-unrest`

This improves graph and search clarity.

## 30-Day Improvement Plan

### Phase 1 (Now)

- Run from [[Session 9 - Ops Index]]
- Use [[Session 9 - GM Command Board]] live
- Keep all quick updates in one place

### Phase 2 (Next Session)

- Add one Dataview dashboard for active hooks and heat
- Add QuickAdd captures for consequences and hooks
- Save one reusable live workspace layout

### Phase 3 (Campaign Standardization)

- Clone this Session 9 ops model into Session 10+
- Introduce identical fields/properties across all session packets
- Keep deep lore in separate refs; keep runbooks intentionally short

## Practical Rule: Runtime vs Reference

- If it must be read during a roll: keep in runbook/board.
- If it is optional depth: keep in reference notes.

This one rule will prevent most future note bloat.
